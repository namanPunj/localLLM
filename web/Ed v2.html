Explain how transformers work in simple terms. I have a CS background but never studied ML.
Ed · 2:14 PM · 2.1s · 387 tok
✓ Searched 6 sources ✓ Read 3 references

Think of a transformer as a conversation between every word and every other word, repeated in layers.

The core trick: attention

For each word, the model asks: "of all the other words around me, which ones should I pay attention to right now?" It then builds a weighted mix of those words and uses that as its new understanding of the current word.

Concretely, every token produces three vectors — Query, Key, and Value:

  • Query — what am I looking for?
  • Key — what do I offer?
  • Value — what's my actual content?

You compute softmax(Q·Kᵀ / √d)·V — basically a similarity-weighted average. Here's it in PyTorch:

scaled_dot_product.py PYTHON 
import torch
import torch.nn.functional as F

def attention(Q, K, V):
    # Q, K, V: (batch, seq, d_model)
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)
    weights = F.softmax(scores, dim=-1)
    return weights @ V

Stack 6–96 of these on top of each other, sprinkle in layer norms and an MLP per layer, and you have GPT.

↳ Follow-up · "What about positional encoding?"
Ed · 2:16 PM

Great catch — attention itself is order-blind. Positional encoding patches that by adding a unique signal to each token based on its position in the sequence. Two common flavors:

  • Sinusoidal (original): fixed sine/cosine waves at different frequencies — elegant, generalizes to longer sequences.
  • Learned / RoPE: position vectors the model learns (or rotation-based, as in LLaMA) — usually performs better in practice.

The position info gets added to the token embeddings before they ever see attention.
